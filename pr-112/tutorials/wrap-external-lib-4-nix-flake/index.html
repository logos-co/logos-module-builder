    Tutorial 4 — An external Nix flake¶

    Goal: wrap a library that is already packaged as a Nix flake exposing a package output (a derivation with lib/ + include/). The builder uses that output directly — no build_command.

    When to use: the library has its own flake.nix that builds it with Nix. This is the cleanest option: the dependency is pinned in your flake.lock, built reproducibly by Nix, and you choose exactly which output to consume.

    Runnable doctest

    The module's blocks below are transcluded from the worked example at tests/fixtures/extlib-doctests/case4-flake-package, built + checked in CI (checks.<system>.extlib-case4-flake-package). The check stands in for the external flake by building libgreet as a derivation and selecting it through the same structured externalLibInputs form shown below.

    How the builder decides¶

    When you point externalLibInputs at a resolved flake package (a Nix derivation), the builder detects it (lib.isDerivation) and uses it as-is — it copies the derivation's lib/ and include/ into your module. If you instead point it at raw source (flake = false), it runs the build_command (Tutorial 3). Same field, two behaviors, chosen by what the input resolves to.

    What the external flake looks like¶

    The library's flake just needs to expose a package per system with lib/ and include/ (illustrative — substitute your real library flake):

    # flake.nix of the external libgreet
outputs = { self, nixpkgs, ... }: let
  forAll = f: nixpkgs.lib.genAttrs
    [ "x86_64-linux" "aarch64-linux" "x86_64-darwin" "aarch64-darwin" ]
    (system: f (import nixpkgs { inherit system; }));
in {
  packages = forAll (pkgs: {
    lib = pkgs.stdenv.mkDerivation {
      pname = "libgreet"; version = "1.0.0";
      src = ./.;
      installPhase = ''
        mkdir -p $out/lib $out/include
        $CC -fPIC -shared -Wl,-soname,libgreet.so -o $out/lib/libgreet.so greet.c
        cp greet.h $out/include/
      '';
    };
    default = self.packages.${pkgs.system}.lib;
  });
};

    metadata.json¶

    Only the name is needed — there is no build_command, because the input is already built:

    metadata.json
    {
  "name": "greetflake",
  "version": "1.0.0",
  "type": "core",
  "category": "integration",
  "description": "Wraps a C library provided by an external Nix flake (package output used directly)",
  "main": "greetflake_plugin",
  "dependencies": [],

  "nix": {
    "packages": {
      "build": [],
      "runtime": []
    },
    "external_libraries": [
      { "name": "greet" }
    ],
    "cmake": {
      "find_packages": [],
      "extra_sources": [],
      "extra_include_dirs": ["lib"],
      "extra_link_libraries": []
    }
  }
}

    flake.nix¶

    Add the library flake as an input and select the output with the structured externalLibInputs form (input + packages):

    flake.nix
    {
  description = "greetflake — Logos module wrapping a library from an external Nix flake";

  inputs = {
    logos-module-builder.url = "github:logos-co/logos-module-builder";

    # The external library, provided as a Nix flake that exposes
    # packages.<system>.lib (a derivation with lib/ + include/).
    greet.url = "github:example/libgreet-flake";
  };

  outputs = inputs@{ logos-module-builder, ... }:
    logos-module-builder.lib.mkLogosModule {
      src = ./.;
      configFile = ./metadata.json;
      flakeInputs = inputs;
      # Structured form: pick which package output of the flake to use. Because
      # the resolved value is a Nix derivation, the builder uses it directly —
      # no build_command needed.
      externalLibInputs = {
        greet = {
          input = inputs.greet;
          packages.default = "lib";
        };
      };
    };
}

    packages.default = "lib" tells the builder to use greet.packages.<system>.lib. If the flake exposes multiple builds (e.g. a portable variant), map them per-variant — packages = { default = "lib"; portable = "lib-portable"; }. If packages.<system>.default is what you want, the shorthand externalLibInputs = { greet = inputs.greet; } also works.

    CMakeLists.txt¶

    EXTERNAL_LIBS greet finds, links, and bundles the flake-provided library — the builder has staged its lib/ + include/ into lib/:

    CMakeLists.txt
    # greetflake — wraps a library provided by an external Nix flake.
cmake_minimum_required(VERSION 3.14)
project(GreetflakeModulePlugin LANGUAGES CXX)

if(DEFINED ENV{LOGOS_MODULE_BUILDER_ROOT})
    include($ENV{LOGOS_MODULE_BUILDER_ROOT}/cmake/LogosModule.cmake)
elseif(EXISTS "${CMAKE_CURRENT_SOURCE_DIR}/cmake/LogosModule.cmake")
    include(cmake/LogosModule.cmake)
else()
    message(FATAL_ERROR "LogosModule.cmake not found")
endif()

file(READ "${CMAKE_CURRENT_SOURCE_DIR}/metadata.json" METADATA_JSON)
string(JSON MODULE_NAME GET ${METADATA_JSON} name)
string(JSON MODULE_VERSION GET ${METADATA_JSON} version)
file(WRITE "${CMAKE_CURRENT_BINARY_DIR}/module_config.h"
    "#pragma once\n"
    "#define MODULE_NAME \"${MODULE_NAME}\"\n"
    "#define MODULE_VERSION \"${MODULE_VERSION}\"\n"
)
include_directories("${CMAKE_CURRENT_BINARY_DIR}")

# The flake's package output (lib/ + include/) is staged into lib/ by the
# builder. EXTERNAL_LIBS greet finds, links, and bundles it.
logos_module(
    NAME ${MODULE_NAME}
    SOURCES
        src/greetflake_interface.h
        src/greetflake_plugin.h
        src/greetflake_plugin.cpp
    EXTERNAL_LIBS
        greet
)

    Plugin code¶

    Same shape as the other tutorials — hello() calls greet_hello() via #include "lib/greet.h":

    src/greetflake_plugin.cpp
    #include "greetflake_plugin.h"
#include "logos_api.h"
#include "lib/greet.h"
#include <QDebug>

GreetflakePlugin::GreetflakePlugin(QObject* parent)
    : QObject(parent)
{
}

GreetflakePlugin::~GreetflakePlugin() = default;

void GreetflakePlugin::initLogos(LogosAPI* api)
{
    // Assign the inherited PluginInterface::logosAPI member — the host reads it
    // directly to dispatch cross-module calls.
    logosAPI = api;
}

QString GreetflakePlugin::hello()
{
    const QString out = QString::fromUtf8(greet_hello());
    emit eventResponse("hello", QVariantList() << out);
    return out;
}

    Build, inspect, run¶

    ws build greetflake                                  # or: nix build
lm methods ./result/lib/greetflake_plugin.so
logoscore -m ./result/lib -l greetflake -c "greetflake.hello()"
# -> "hello from libgreet"

    Notes¶

    • The doctest builds libgreet as a Nix derivation, exposes it as packages.<system>.greetlib, and selects it with { input = …; packages.default = "greetlib"; } — exactly the structured form above. It then asserts the flake-built library is bundled and linked (otool -L / readelf -d).
    • Because Nix builds the dependency, its transitive runtime libraries are handled by the closure; the RPATH (@loader_path/$ORIGIN) resolves the bundled libgreet next to the plugin.
